Write-up


Intro

This dataset was taken from Chicago Police Departments's CLEAR program. In this database, there are all instances of crime reported in Chicago from 2001 to the present day. For each report, we looked into the year and amount of each type of crime committed (eg. theft, burglary, etc). Our goal was to visualize the crime count over time, categorized by the type.

Design Decisions

We ended up picking a line graph as is it is quite effective in comparing data over time. Line graphs would also make it easy for the user to compare trends. We considered taking other subsets of data and visualize using bar charts; one approach we thought of was comparing districts and crime rates. We ultimately decided to pick a line graph since we wanted to show trends over time.

Required features:
We allow the user to highlight each line (type) on the graph to see more info on how many reports were there on that year. (Category 2) We also allow the user to filter certain types of crimes by using checkboxes at the bottom. (Category 1)

Discussion:
In particular, we felt that it would be most helpful for users to be able to compare trends of different crimes over time, and also compare them with the overall trend in all crimes. For this, we decided to add checkboxes for users to select the crimes they wanted to view, which also helps with decluttering the data. We implemented a scaling y-axis that adjusts based on the max of the data currently selected, so this allows users to more clearly observe trends within crimes that are less frequent without sacrificing the ability to analyze all of the different crime types. Along similar lines (get it? ðŸ˜Š), we also made sure our tooltip provided the crime count number at an appropriate scale so as to not include too much information (the exact total) while also still providing a number in the thousands to show the extra detail.

For more specific elements of the chart interface, considering the issue that dual axis charts have of the drastic impact of axis scales, we decided that the scaling y-axis would help mitigate this because of the variety of views that the users can generate. This would help the user get a more general sense of the crime trends in the context of several other categories to avoid any dramatic bias.

We also decided to add a tooltip selection (elaboration) by considering the user's desire to evaluate the information, as in Bloom's taxonomy for learning. Showing the exact counts within each line helps the user put its magnitude both within the context of the same crime and different crimes, while cross-comapring within different years as well if they would like.

Reflections:
Overall, we found this assignment to be challenging to implement because of how we had to incorporate our intended design using the very high-level methods that d3 provides. However, we found existing documentation to be very helpful in figuring out issues. We believe we were successful in creating a highly customizable, clean visualization that effectively showcases visual comparisons of trends in crime.
